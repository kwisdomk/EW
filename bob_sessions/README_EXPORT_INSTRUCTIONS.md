## 🔧 Step-by-Step Export Process

### Step 1: Open Bob History
1. In Bob IDE's chat interface, click **"Views and More Actions"** (three dots menu)
2. Select **"History"** option
3. The History tab will appear

### Step 2: Select Workspace
1. Confirm you are in the correct project workspace: **EW (EpidemiWatch)**
2. If you have tasks from multiple workspaces, select **"All"** to view all tasks

### Step 3: Export Each Task Session

For **EACH** task related to your EpidemiWatch project:

1. **Select the task** from the history list (it will open in chat panel)
2. **Click the task header** to view the task session consumption summary
3. **Take a screenshot** of the consumption summary
   - Save as: `bob_sessions/SCREENSHOT_TaskName_consumption.png`
4. **Click the "Export task history" icon** (download icon)
   - This downloads a markdown file
5. **Move the downloaded .md file** to `bob_sessions/` folder
6. **Rename appropriately** (see naming convention below)

### Step 4: Repeat for All Tasks
Export all tasks related to:
- Backend foundation
- Detection engine
- Frontend development
- API integration
- Any other EpidemiWatch work

## 🎯 Why This Matters for Judging

The judges need to see:
1. **How much you used Bob** - consumption metrics
2. **What Bob generated** - exported task history
3. **Quality of prompts** - visible in task history
4. **Code generated by Bob** - documented in exports
5. **Time saved** - evident from consumption data

This documentation proves your IBM Bob usage and helps judges evaluate your project fairly.

## 💡 Tips

1. **Export immediately after each session** - Don't wait until the end
2. **Take clear screenshots** - Ensure consumption metrics are visible
3. **Name files descriptively** - Makes it easy to match screenshots with exports
4. **Keep originals** - Don't modify the exported markdown files
5. **Document everything** - If Bob helped, export that session
